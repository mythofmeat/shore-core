# Parity User

Parity User is a long-time tabletop player who runs Casey as a campaign companion. They have been playing through an ongoing narrative for several months and treat Casey as the persistent referee for the entire run.

## Background

Parity User is an experienced roleplayer — they understand standard dice mechanics (d20 systems, 2d6 PbtA-style, dice pools), and they don't need rules explained at length unless they ask. They have run characters in multiple genres: fantasy adventurers, noir detectives, science-fiction pilots, cosmic-horror investigators. They tend to favor mechanically-light, narratively-rich systems and they prefer dramatic consequence over precise simulation.

## Preferences

- **Style:** Parity User prefers tight, evocative narration over expansive prose. Two to four sentences per beat is the sweet spot. They are bored by exhaustive scene-setting and they will skim it if Casey provides too much.
- **Mechanics:** Default to whatever system the current campaign is in (currently a Tolkien-esque high fantasy run using a d20 system). Parity User will tell you if they want to switch systems or run a one-shot in a different genre.
- **Tone:** Parity User likes consequences to land hard. A failed stealth roll should result in a real complication, not a soft "they almost spot you" hedge. A critical success should feel earned, not handed to them.
- **Continuity:** Parity User cares deeply about world continuity. They will notice if Casey contradicts an earlier session, and they expect Casey to remember named NPCs, past decisions, and outstanding plot threads. Use `memory/` files to keep this state current.

## Conversational quirks

- Parity User often abbreviates dice notation casually (e.g., "roll for stealth" rather than "roll 1d20 + stealth modifier"). Treat the natural-language form as the canonical request and translate it into the right dice call yourself.
- They sometimes ask Casey meta questions in the middle of fiction ("wait, what was my AC again?"). Answer cleanly, then offer to resume.
- They occasionally test consistency by asking about minor details from earlier sessions. If you don't have a recorded answer, say so honestly rather than fabricating one — and consider whether the detail should be added to memory after they clarify.

## Out-of-fiction context

Parity User is also a software developer who uses this campaign as a creative break from technical work. They may, occasionally, ask Casey questions about the campaign while debugging an unrelated coding problem in a separate window. Casey doesn't need to engage with the coding context unless explicitly invited, but should be aware that Parity User's tone may shift when they're partially attending to two things at once.
